SECTION ORDERING
  The high-signal blocks (capstone → cert → prereq) at the top work well individually.
  When multiple are present (e.g., C824 capstone + prereq), the stack feels justified.
  When only one is present (e.g., C178 cert, D426 no top blocks), the page opens cleanly.

VARIANT DISPLAY POLICY (NOT YET RESOLVED)
  The prototype over-shows all variants. For C169 (3+3 panels), this creates a long,
  complex page. For C165 (description collapsed, competencies may still be 3 panels),
  the result may feel inconsistent.
  Questions to resolve:
    - When should descriptions collapse vs. show?
    - When should competencies collapse vs. show?
    - Should "collapse" be a UI toggle (show more) or a data-normalization decision?
    - Is "cosmetically identical" the right criterion, or should it be "same source programs"?

PREREQ REDUNDANCY IN DESCRIPTIONS
  For C480 and C170, the guide description text itself mentions the prerequisite.
  Showing a Requires block AND having the prereq in the description text creates
  duplication. Options: (a) strip prereq language from description display, (b) note
  "see above", (c) leave as-is (reviewable duplicate).

PROGRAM-CONTEXT GAP IN PREREQS
  C169's BSDA prereq (C859) does not apply to BSCS/MSCSUG students.
  Displaying the prereq without program context is potentially misleading.
  This is the clearest argument for program-context framing in prereq blocks.

CERT BLOCKS
  All three cert cases (C178, C480, C176) use the emerald block.
  The emerald is distinctive and high-signal. No concern about over-prominence.
  The cert confirmation footnote (N programs, source program list) is useful provenance.

REVERSE PREREQ BLOCK
  D426's violet "Prerequisite For" block is the only instance in the cohort.
  It's a clean, useful piece of navigation. Whether violet vs. blue distinction
  is meaningful needs review.

SPARSE FALLBACK (C216)
  The fallback notice is explicit and honest. The capstone callout (title-only)
  adds minimal value but surfaces the inference transparently.
  The most important question for C216: is the catalog description adequate? If yes,
  the production page is fine. If the catalog desc is also thin, a different approach
  is needed.

COLLEGE FIELD ANOMALY
  C176 shows "College of Health Professions" which appears incorrect for this IT course.
  This anomaly is visible in the prototype header. The prototype is correctly surfacing
  data quality issues that would exist in production too.
