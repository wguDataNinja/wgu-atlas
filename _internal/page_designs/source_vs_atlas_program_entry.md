## Purpose of this doc

This document preserves the before-vs-after analysis between:

1. the raw WGU catalog degree entry
2. the Atlas program detail page

Its job is to make explicit what Atlas changes, not just what Atlas contains.

This is a product-argument artifact:
- useful for homepage planning
- useful for positioning
- useful for explaining why the site matters
- useful for avoiding weak "just a nicer catalog" framing

---

## Why this comparison matters

The raw catalog is the source authority for much of Atlas's degree information, but the source presentation is fragmented and limited.

A student trying to understand one degree from the raw catalog must navigate across multiple sections to reconstruct a complete picture:
- program entry
- program outcomes
- course descriptions
- external official resources not attached in the catalog

Atlas consolidates and restructures that experience.

The value here is not cosmetic. It is structural.

## What the raw catalog entry gives the student

The raw catalog degree entry provides:

- program title
- one short description paragraph
- flat curriculum table
- term numbers in a final table column
- CU count per course
- one total line at bottom

It does **not** provide a consolidated degree understanding experience.

### Important structural limitations
- no explicit term-group section headers
- no clickable course navigation
- no inline outcomes
- no attached official supporting resources
- no history/change context
- no sidebar or related links
- no clear program-code identity beyond the terminal total row
- no comparison affordance

### Visual character of the source
- dense
- tabular
- disclosure-oriented
- static
- flat
- multi-page
- hard to scan as a whole

---

## What Atlas gives the student on the program page

Atlas program pages consolidate and restructure the degree information into a usable, navigable surface.

Observed BSCS page includes:

- clear degree identity block
- code, status, and current CU signal
- school association as a live link
- official description surfaced cleanly
- explicit degree history
- learning outcomes inline
- term-grouped course roster
- clickable course codes
- curated relevant resources in sidebar
- repeated provenance markers

This creates one-page degree understanding rather than multi-section reconstruction.

## Structural differences

### 1. Flat table vs structured page
Raw catalog:
- title + paragraph + one long table

Atlas:
- distinct sections with headings
- different content types clearly separated
- metadata treated differently from narrative and from roster data

### 2. Term numbers vs term grouping
Raw catalog:
- term is just a numeric column in a flat table

Atlas:
- each term is visually grouped as its own roster block

This matters because Atlas turns the degree from a row list into a readable plan structure.

### 3. Snapshot vs contextual page
Raw catalog:
- one static program snapshot

Atlas:
- current degree identity plus history plus outcomes plus linked courses plus resources

### 4. Dead text vs navigable graph
Raw catalog:
- course titles are inert table cells

Atlas:
- course codes are navigation targets to course detail pages

### 5. Isolated entry vs connected information surface
Raw catalog:
- degree entry is isolated from outcomes, course descriptions, and official resources

Atlas:
- those layers are integrated or attached directly

## Student usability differences

### Degree description
Raw catalog:
- one paragraph above table

Atlas:
- same content, but separated into a dedicated section with source labeling and clearer reading treatment

### Roster comprehension
Raw catalog:
- continuous table
- terms only inferred from final column
- page break interrupts the experience

Atlas:
- explicit term headers
- compact readable tables per term
- easier to scan and remember

### Course understanding
Raw catalog:
- a student must leave the degree entry and manually search the course descriptions section for each code

Atlas:
- every course code is a direct link to a course page

### Outcomes access
Raw catalog:
- outcomes are 90+ catalog pages away in a separate section

Atlas:
- outcomes are on the same degree page

### Resource discovery
Raw catalog:
- no curated official links attached to the degree entry

Atlas:
- a Relevant Resources sidebar surfaces official next-step materials

### Historical understanding
Raw catalog:
- no first-offered date
- no CU history
- no college-name history
- no current/retired framing beyond implicit edition state

Atlas:
- degree history section makes these signals visible

## What this means for the student task

### In the raw catalog, to understand one degree, a student may need to:
1. find the degree entry
2. read the paragraph
3. read the continuous table
4. note course codes
5. jump to course descriptions for any course they want to understand
6. jump to outcomes later in the catalog
7. separately search the WGU website for guides or supporting materials

### In Atlas, the student can:
1. open one degree page
2. read description
3. review outcomes
4. scan term-grouped roster
5. click into any course
6. see official resources immediately
7. see useful change/history context

This is a large reduction in research friction.
