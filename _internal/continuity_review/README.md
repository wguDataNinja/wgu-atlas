# Continuity Review Module

## Module purpose

Establish and maintain a lightweight review method for degree continuity cases in WGU Atlas. This module helps the team decide whether continuity deserves more product weight, what a lightweight continuity surface should look like, and whether course continuity should remain deferred.

## In-scope questions

- What is the minimum viable format for reviewing degree continuity cases?
- Which continuity events provide clear value for current students?
- How can we review continuity cases without building heavy UI systems?
- What patterns emerge from approved vs. pending vs. excluded events?

## Out-of-scope for now

- Building full continuity viewer features
- Course-level continuity review
- Complex lineage visualization systems
- Large-scale production review batches
- Cross-program relationship mapping

## Current state

- Module workspace initialized
- Planning memo drafted defining review method approach
- 17 approved events identified for potential review
- 3 pending cases requiring HITL decisions
- Review method format selected (compact text cards)

## Locked assumptions

- Continuity is supporting context, not main product identity
- Review method must be lightweight and low-maintenance
- Focus on degree-level continuity only
- Human review required for ambiguous cases
- Compact text format preferred over complex visualizations

## Open questions

- What is the optimal batch size for initial validation?
- Which specific events should be included in the first review batch?
- How should HITL decisions be documented and tracked?
- What criteria determine when to expand review scope?
